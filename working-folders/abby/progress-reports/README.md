# Weekly progress reports

One markdown file per week, due **every Friday by EOD**. Push it as part of your normal commit flow — no separate PR needed unless your manager asks for one.

## Filename format

```
YYYY-MM-DD.md
```

Use the **Friday** of that week (the date you're submitting). Example: `2026-06-13.md`.

## Format

Copy `_template-week.md` (below) for each new week. Keep it short — bullets, not essays. Your future self and your manager skim these at evaluation time; nobody wants to wade through prose.

## The template

Save the following as `_template-week.md` in this folder, then duplicate it each Friday:

```markdown
# Week of YYYY-MM-DD

## What I shipped this week

- [ ] Concrete output #1 (link to PR / Linear ticket / wiki note)
- [ ] Concrete output #2

## What I learned

- One thing that clicked. Specific, not vague.

## What I'm blocked on

- Blocker + who can help unblock it. Be honest — "nothing" is fine if true, but vague "moving forward" is not.

## Next week

- [ ] Top priority
- [ ] Stretch goal (if any)

## Time and budget

- **Hours worked this week:** N
- **Notable agent spend (Paperclip $):** $N (or "n/a")
```

## Tips

- **Link to evidence.** Each "what I shipped" line should link to a PR, ticket, doc, or commit. Words without links rot.
- **Write it Friday afternoon, not Sunday night.** It's faster and more accurate while the week is fresh.
- **Use these for your 1:1 with Arianna.** She reads them before the meeting; come prepared to discuss anything flagged.
